Auction engine 4 areas
Auction mechanics P1
›Sealed-bid first-price model
›Configurable window — 12h, 24h, or 48h
›Fixed daily batch start time at 09:00 CET
›Optional reserve price — hidden from all dealers
›Minimum bid increment of 500 PLN
›Upward revision only — max 3 per dealer per listing
›Sniping prevention — +5 min on late bid, max +30 min
›Zero-bid outcome — extend 24h once or relist
Bid integrity and security P1
›Cryptographic server-side timestamp on each bid
›Session verified server-side on every bid action
›Rate limiting — 10 bid actions per minute per dealer
›Immutable audit trail with timestamp, IP, amount
›Fraud detection with anomaly alerts
›Mid-auction dealer suspension by ops (non-disruptive)
›Idempotent bid API — zero data loss tolerance
Real-time UI P2
›WebSocket live updates — countdown and bid count
›Graceful fallback to 30-second polling
›Two-step bid confirmation — review then confirm
›Outbid push notification and in-app alert
›1-hour closing alert via push and SMS
›Full bidding functionality on mobile browser
Post-auction workflow P1
›Auction closed — bids locked, seller notified
›Offer accepted — winner gets full seller contact details
›Losing dealers automatically notified
›Collection scheduling with .ics calendar invite
›Sale complete — dealer invoiced for success fee
›Offer expired at 48h — seller prompted to relist
›Dispute raised — ops mediation with 5-day SLA
